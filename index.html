MCP Client Boot Starter

개요

MCP(Model Context Protocol) Client Boot Starter는 Spring Boot 애플리케이션에서 MCP 클라이언트 기능을 위한 자동 구성을 제공합니다. 동기 및 비동기 클라이언트 구현을 지원하며, 다양한 전송 옵션과 클라이언트 관리 기능을 제공합니다.

주요 스타터

1. 표준 MCP 클라이언트

spring-ai-starter-mcp-client

STDIO 및/또는 SSE 전송을 통해 MCP 서버에 연결

2. WebFlux 클라이언트

spring-ai-starter-mcp-client-webflux

WebFlux 기반 SSE 전송 구현 사용

구성 속성

  • 공통 속성 접두사: spring.ai.mcp.client
  • 구성 가능한 설정:
    • 클라이언트 활성화/비활성화 상태
    • 클라이언트 이름 및 버전
    • 요청 타임아웃
    • 클라이언트 타입 (SYNC 또는 ASYNC)
    • 전송별 구성

예제 구성

spring:
  ai:
    mcp:
      client:
        enabled: true
        name: my-mcp-client
        type: SYNC
        sse:
          connections:
            server1:
              url: http://localhost:8080

주요 기능

  • 동기 및 비동기 클라이언트 타입 지원
  • 콜백 인터페이스를 통한 광범위한 클라이언트 커스터마이징
  • 다중 전송 지원 (Stdio, SSE HTTP, SSE WebFlux)
  • Spring AI의 도구 실행 프레임워크와의 통합

MCP Server Boot Starter

개요

MCP Server Boot Starter는 Spring Boot 애플리케이션에서 MCP 서버 설정을 위한 자동 구성을 제공합니다. 동기 및 비동기 작동 모드를 지원하며, 다양한 전송 계층 옵션을 제공합니다.

스타터 옵션

1. 표준 MCP 서버

spring-ai-mcp-server-spring-boot-starter

STDIO 전송 - 커맨드라인 및 데스크톱 도구에 적합

2. WebMVC 서버 전송

spring-ai-starter-mcp-server-webmvc

Spring MVC를 사용한 SSE - HTTP 기반 전송 제공

3. WebFlux 서버 전송

spring-ai-starter-mcp-server-webflux

반응형 SSE - Spring WebFlux를 사용한 반응형 전송

주요 기능

  • 도구 관리
  • 리소스 관리
  • 프롬프트 관리
  • 완성 관리
  • 변경 알림 지원

예제 구성

spring:
  ai:
    mcp:
      server:
        name: webmvc-mcp-server
        version: 1.0.0
        type: SYNC
        instructions: "날씨 정보를 제공하는 서버"

사용 예제

@SpringBootApplication
public class McpServerApplication {
    @Bean
    public ToolCallbackProvider weatherTools(WeatherService weatherService) {
        return MethodToolCallbackProvider.builder()
            .toolObjects(weatherService)
            .build();
    }
}

MCP Helpers

개요

Spring AI 애플리케이션과 Model Context Protocol 통합을 위한 기초 지원을 제공합니다. Spring AI의 도구 시스템과 MCP 서버 간의 통신을 활성화하며, 동기 및 비동기 작업을 모두 지원합니다.

주요 컴포넌트

1. ToolCallback 유틸리티

  • MCP 도구를 Spring AI의 도구 인터페이스에 적응시킴
  • 동기 및 비동기 실행 모두 지원
  • 도구 콜백을 도구 명세로 변환
  • MCP 클라이언트에서 도구 콜백 검색

2. McpToolUtils

  • 도구 콜백을 도구 명세로 변환
  • 동기 및 비동기 도구 명세 모두 지원
  • MCP 클라이언트에서 도구 콜백 추출 가능

3. Native Image 지원

  • McpHints 클래스가 GraalVM native image 힌트 제공
  • MCP 스키마 클래스에 대한 리플렉션 힌트 자동 등록

코드 예제

동기 도구 콜백 생성:

McpSyncClient mcpClient = // MCP 클라이언트 획득
Tool mcpTool = // MCP 도구 정의 획득
ToolCallback callback = new SyncMcpToolCallback(mcpClient, mcpTool);

비동기 도구 콜백 생성:

McpAsyncClient mcpClient = // MCP 클라이언트 획득
Tool mcpTool = // MCP 도구 정의 획득
ToolCallback callback = new AsyncMcpToolCallback(mcpClient, mcpTool);

전체 요약

이 문서는 동기 및 비동기 작업, 다양한 전송 메커니즘, 그리고 광범위한 커스터마이징 옵션을 지원하여 개발자가 강력한 AI 기반 애플리케이션을 구축할 수 있도록 합니다.
